# 웅진 자막 검수 도구 (어디서든 접속)

설치 프로그램 없이 인터넷 주소(URL)로 어디서든 여는 검수 도구입니다.
사람이 꼭 해야 하는 2배속 시청·실제 수정만 남기고, 나머지(링크 관리·오류 사전탐지·진도/정산·보고)는 자동입니다.
완료/보류를 누를 때마다 **구글 시트**에 자동으로 쌓이고, 다른 기기에서도 그 시트에서 이어서 할 수 있습니다.

이 폴더에는 두 개가 들어 있습니다.
- `index.html` — 도구 본체 (Vercel에 올릴 파일)
- `구글시트_연결코드.gs` — 구글 시트에 한 번만 붙여넣는 연결 코드

---

## 1단계 — Vercel에 올려서 주소 만들기 (약 3분)

깃허브 + Vercel을 쓰는 가장 쉬운 길입니다.

1. 깃허브에서 새 저장소(repository)를 만든다. 예: `jamak-review` (Public/Private 아무거나)
2. 이 폴더의 `index.html`을 그 저장소에 올린다.
   - GitHub Desktop 사용 시: 저장소 폴더에 `index.html`을 복사 → Commit → Push
   - 웹에서 바로 올릴 때: 저장소 화면 → `Add file` → `Upload files` → `index.html` 끌어다 놓기 → Commit
3. [vercel.com](https://vercel.com) 로그인 → `Add New…` → `Project`
4. 방금 만든 깃허브 저장소를 `Import`
5. Framework Preset 은 **Other** 로 두고 (빌드 설정 건드릴 것 없음) → `Deploy`
6. 끝나면 `https://jamak-review.vercel.app` 같은 주소가 나온다. **이 주소를 어디서든 열면 됨.**

> 나중에 도구를 고치고 싶으면 `index.html`만 다시 Push 하면 Vercel이 자동으로 새로 배포합니다.

---

## 2단계 — 구글 시트 연결 (약 2분, 한 번만)

데이터를 한곳에 모으고 다른 기기에서 이어하려면 필요합니다. (안 해도 도구는 동작하고, `엑셀로 받기`로 결과를 뽑을 수 있습니다.)

1. 주소창에 `sheets.new` → 새 구글 시트 생성
2. 상단 메뉴 **확장 프로그램 → Apps Script**
3. 보이는 코드를 전부 지우고, `구글시트_연결코드.gs` 내용을 통째로 붙여넣고 저장
4. 우측 상단 **배포 → 새 배포**
   - 유형(톱니바퀴): **웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자**
   - **배포** → 권한 검토/허용
5. 나온 **웹 앱 URL**(`.../exec`로 끝남) 복사
6. 도구 화면의 **설정 → 구글 시트 주소** 칸에 붙여넣기

연결되면 완료/보류를 누를 때마다 시트에 `검수일정표 · 오류목록 · 정산표`가 자동으로 채워집니다.
다른 컴퓨터/노트북에서 같은 도구 주소를 열고 같은 시트 주소를 넣은 뒤 **시트에서 불러오기**를 누르면 하던 작업이 그대로 이어집니다.

---

## 매일 쓰는 법 (요약)

1. 도구 주소 열기 → (처음엔) 일정표 붙여넣기 / 구글 시트 주소 넣기
2. 영상 고르기 → **링크 복사** → **웅진 사이트 열기** → 붙여넣고 검수
3. 자막을 도구에 붙여넣으면 알려진 오타를 짚어줌 (참고용)
4. 웅진에서 고치고 **저장 → CDN Upload**
5. 도구에서 **완료 처리** (자동으로 시트에 기록)
6. 보고는 그 구글 시트 주소를 공유하면 끝

## 한계 (정직하게)

- 자막을 도구에 **붙여넣는 한 단계**는 남습니다. 웅진 자막을 자동으로 읽어오려면 별도 서버가 필요해 일부러 넣지 않았습니다(복사 한 번이 가장 안 깨집니다).
- 영상 2배속 시청·실제 수정은 사람이 직접 해야 하는 일이라 자동화 대상이 아닙니다.
- 시트로 보낼 때 브라우저 보안상 "보냈다"는 표시만 뜹니다. 처음 한 번은 시트가 실제로 채워지는지 눈으로 확인하세요. 안 되면 거의 항상 4번의 "액세스: 모든 사용자" 설정 또는 주소 끝 `/exec` 문제입니다.
